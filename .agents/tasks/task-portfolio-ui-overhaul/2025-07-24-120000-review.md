# Portfolio UI overhaul: navbar simplification, dark theme fix, project detail page, horizontal tabs, skills removal, contact redesign

Six related changes strip the cyber aesthetic from the portfolio, fix dark theme coverage, introduce a full-page project detail overlay, restructure the About section tabs to horizontal/swipeable, remove the redundant Skills section, and rebuild the Contact form with validation and interactive social cards. The work consolidates project data into a shared module, unifies `bg-white` usages to `bg-surface`, and adds CSS variable overrides for glassmorphism, scrollbar, noise-overlay, and aurora backgrounds in dark mode.

**Watch for:** Body scroll bleeds through when the ProjectDetail overlay is open (confirmed). The error state in Contact uses hardcoded `bg-red-50`/`border-red-200` that won't adapt to dark theme (confirmed). Orphaned locale keys (`sections.projects.modal.*`) remain in all three locale files (confirmed). Swipe gesture on About tabs may conflict with horizontal page scroll on touch devices (likely).

## High-level view

The navbar loses its cyber-themed decorations (UPLINK_ON, ping animations, `//`-prefixed labels) and gains a flat mobile menu with direct nav links, inline theme toggle, and language switcher. The simplification removes the tab-switcher pattern (navigation/settings) that previously required two extra components.

Dark theme coverage expands from five CSS variable overrides to a full set including `--color-stone`, glassmorphism classes, scrollbar colors, noise-overlay blend mode, and aurora radial gradients. All section components replace `bg-white` with `bg-surface`, but the error state in Contact and the NotFound section still use Tailwind color literals that won't respond to the CSS variable scheme.

The Projects section extracts its data to `src/data/projects.ts`, displays a 2-column mobile grid with compact cards (detail text hidden), and lifts project selection up to `App.tsx` which renders `ProjectDetail` as a fixed z-50 overlay. The overlay does not lock body scroll, so Lenis smooth-scroll on the page beneath continues to fire.

The About section switches from a vertical 3+9 grid layout to a full-width horizontal tab bar with a framer-motion `drag="x"` swipe gesture attached to the content container. The content itself doesn't translate on swipe — only the `onDragEnd` handler switches tabs after a threshold — which means visual feedback during the drag is limited to elastic rubber-banding rather than a carousel slide.

The Contact form adds field validation (blur-triggered, with inline error messages), a progress indicator (3-dot bar), floating labels via `peer-placeholder-shown`, character counter for messages, a proper error state with retry/back, and celebration ring animations on success. The `isSubmitting` ref prevents rapid double-submit.

2. **Error state uses non-theme colors** — `bg-red-50` and `border-red-200` in Contact's error state are Tailwind palette colors that don't respond to `[data-theme='dark']` overrides. Replace with theme-aware equivalents or add explicit dark-mode overrides.

## ProjectDetail overlay and scroll bleed

`ProjectDetail` renders as a `fixed inset-0 z-50` div with its own `overflow-y-auto`. Neither `App.tsx` nor `ProjectDetail` applies `overflow: hidden` to the document body or disables the Lenis smooth-scroll instance. On touch devices the user can still scroll the underlying page through momentum, and on desktop a scroll wheel event that passes beyond the overlay's scrollable bounds will propagate to the main content.

The keyboard escape handler (`useEffect` with `keydown` → close on Escape) that existed in the old `Projects.tsx` modal was removed and not ported to `ProjectDetail`. This is an accessibility regression — keyboard-only users cannot dismiss the overlay without tabbing to the back button.

```tsx
// App.tsx – no effect to lock scroll when selectedProject is truthy
<AnimatePresence mode="wait">
  {selectedProject && (
    <Suspense ...>
      <ProjectDetail project={selectedProject} onBack={() => setSelectedProjectId(null)} />
    </Suspense>
  )}
</AnimatePresence>
```

## Dark theme gaps in Contact error state

The error state renders `bg-red-50 border border-red-200` — Tailwind static colors that remain light red regardless of theme. In dark mode this produces a jarring bright pink circle on an otherwise dark surface. The fix applied everywhere else in this diff (replace with `bg-surface` or theme-aware opacity) was not applied here. Similarly, `NotFound.tsx` retains `bg-white` on its compass icon wrapper (noted in FEAT-002 findings but not addressed).

## About tab swipe mechanics

The swipe wraps the tab content card in a `motion.div` with `drag="x"` and `dragConstraints={{ left: 0, right: 0 }}`. The constraints snap the element back to origin on release, and `onDragEnd` checks `info.offset.x` against a 50px threshold to decide whether to switch tabs.

- During the drag, the content visually deflects horizontally (elastic rubber-band), which looks like a failed scroll rather than a deliberate tab switch gesture.
- The threshold of 50px is low enough that accidental horizontal drift during vertical scroll could trigger a tab switch on mobile.
- `touch-pan-y` is applied to the drag container, which should help the browser prioritize vertical scroll, but framer-motion's drag system may still intercept touch-start events.

## Orphaned locale keys and dead code

The `sections.projects.modal.*` keys (`overview`, `challenge`, `solution`, `architecture`, `launchSite`, `closeLabel`) are present in all three locale files but unreferenced in any component. The `TableOfContents` component file remains on disk and is imported nowhere — dead code after the Navbar rewrite.
